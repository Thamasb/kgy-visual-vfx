# KGY VISUAL 

![KGY VISUAL](https://res.cloudinary.com/duxsycizt/image/upload/v1674768796/kgy-visual/responsive_c3wxvo.png) 


KGY VISUAL is a Visual Effect Artist's website. The business owner and VFX Artist is Gyula Kunsagi, this website is built for him. On this wesite he introduces himself and the services what he can offer, presents his references and gives the opportunity to contact him through the site.

# User Experience

## Project Goals

- The website presents the Visual Effect Artist, his services, references and contact informations.
- The design is clear and simple.
- Offers the customers the opportunity to contact with the VFX Artist.

## User Stories

- As a customer I want to be able to navigate easy at the website.
- As a customer I want to be able to find the business name.
- As a customer a I want to be able to find the navigation bar.
- As a customer I want to able to read more about the VFX Artist's Service.
- As a customer I want to be able to find the VFX Artist's references.
- As a customer I want to be able to watch the videos on the website.
- As a customer I want to contact with the VFX Artist.
- As a customer I want to be able to find more information about the VFX Artist.

- As a customer I want to be able to navigate easy at the website.
    - Every information on one page and the navigation bar helps to find the relevant information.

    - The business name is placed at the header (at the top of the page) and also at the footer (at the bottom of the page)

    - The navigation bar help to find the relevant information.
- As a customer I want to be able to watch the videos on the website.

- As a customer I want to be able to find more information about the VFX Artist.
    - At the footer (bottom of the page) it is easy to find the relevant social network links.

# Deployment

This website was developed using [GitPod](https://gitpod.io/workspaces) which was then committed and pushed to GitHub using the GitPod terminal.

You can clone or forked this repository from [GitHub.com](https://github.com/) to make a local copy.

#### [Cloning](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository)
1. On [GitHub.com](https://github.com/) navigate to the main page of the repository.
2. Above the list of files, click Code.
3. Copy the URL.
4. Open Git Bash.
5. Change the current working directory to the location where you want the cloned directory.
6. Type git clone, and then paste the URL you copied earlier.
7. Press Enter to create your local clone.


#### [Forking](https://docs.github.com/en/desktop/contributing-and-collaborating-using-github-desktop/adding-and-cloning-repositories/cloning-and-forking-repositories-from-github-desktop)
1. In the File menu on [GitHub.com](https://github.com/)(top of the repostory) click to the 'Setting' button.
2. At the left side of the page in the 'Genereal" menu click 'Pages'.
3. Under Branch click the dropdown menu called 'None' and select 'main' / 'Master Branch'
4. Click 'Save'
5. The page automatically generate a link to your website.
